## Riesgos y notas

- La seccion `Vouchers` mejoro mucho, pero todavia no esta cerrada funcionalmente:
  - no existe detalle de lote
  - la emision todavia puede simplificarse mas
  - faltan acciones de dominio como revocar o reenviar
- Parte de la pantalla sigue usando una carga completa de vouchers para armar opciones y algunos KPIs; la experiencia principal ya esta backend-driven, pero la optimizacion no esta terminada
- Los errores `relation "users" does not exist` y `relation "vouchers" does not exist` quedaron descartados como bloqueo actual del codigo; la base local ya tiene el esquema levantado
- No hubo validacion manual exhaustiva de toda la UX del dashboard, solo compilacion y chequeo puntual de endpoints
