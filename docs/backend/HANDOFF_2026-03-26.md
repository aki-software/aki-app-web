## Estado funcional actual

### Flujo 1. Test individual
Comportamiento actual:

- un usuario `PATIENT` puede completar el test sin login admin;
- la sesión se guarda;
- la sesión queda visible desde el panel admin;
- si no hay terapeuta ni voucher, se asigna ownership a un usuario owner interno configurable.

### Flujo 2. Test con código/voucher
Comportamiento actual:

- se puede crear voucher desde admin;
- se puede validar voucher disponible;
- el paciente hace el test usando `voucherCode`;
- la sesión queda vinculada al voucher;
- la sesión hereda ownership del comprador del voucher;
- el voucher queda marcado como usado.
