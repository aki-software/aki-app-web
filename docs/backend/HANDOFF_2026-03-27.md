# Handoff 2026-03-27

## Estado del día

Se cerró una segunda fase importante de ownership, permisos y lenguaje de dominio para vouchers y sesiones.

Quedó resuelto que:

- el ownership principal de vouchers y sesiones es institucional;
- un terapeuta particular se modela como institución privada;
- el backend usa `owner` como lenguaje de dominio principal;
- la persistencia física quedó alineada con `owner_*`;
- los permisos de sesiones quedaron más seguros;
- el flujo institucional fue probado de punta a punta;
- el flujo de terapeuta particular como owner institucional también fue probado.

## Estado funcional actual

## Regla MVP consolidada

- el owner principal es institucional;
- el terapeuta es responsable operativo;
- el paciente no tiene historial web en el MVP;
- el paciente interactúa sólo desde Android;
- la institución tiene un solo terapeuta por ahora;
- los vouchers los crea el admin de plataforma;
- no existe todavía subasignación interna entre múltiples terapeutas.

## Qué ya se puede considerar sólido

- ownership de vouchers;
- ownership de sesiones;
- persistencia enriquecida de resultados;
- vínculo voucher -> sesión;
- error útil cuando un voucher no está disponible;
- naming consistente de dominio `owner`.
